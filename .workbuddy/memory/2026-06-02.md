## CORS 图片跨域修复

### 问题
前端加载 `http://39.106.193.56:8080/uploads/...` 和 `https://admin.xiaoyangxiaozhang.xyz/uploads/...` 的图片时被浏览器拦截：
`No 'Access-Control-Allow-Origin' header is present on the requested resource`

### 解决方案：Nitro 图片代理 + 数据层改写

#### 新建文件
- **`server/routes/proxy-image.get.ts`** — Nitro 代理路由，接收 `?url=` 参数，从远端拉取图片并返回（含 `Access-Control-Allow-Origin: *` 和强缓存头）
- **`app/composables/proxyImage.ts`** — 两个工具函数：
  - `proxyImageUrl(url)` — 将外部图片 URL 转为 `/proxy-image?url=...` 同源代理
  - `proxyHtmlImages(html)` — 对 HTML 中的 `<img src>` 进行代理替换

#### 修改的数据层
| 文件 | 修改内容 |
|------|---------|
| `app/composables/articleDisplay.ts` | `mapArticleCard` 中的 `cover` 字段走代理 |
| `app/composables/commentDisplay.ts` | `mapSingleComment` 中的 `avatar` 字段走代理 |
| `app/composables/commentRenderer.ts` | `renderCommentContent` 中的 `src` 字段走代理 |
